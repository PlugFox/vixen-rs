# SEO / Meta (Vixen website)

**Source:** [Google Search Central docs](https://developers.google.com/search/docs).

## Scope

Vixen's website has **two audiences** with very different SEO requirements:

- **Admin dashboard** (`/app/*`, authenticated): `noindex`, never crawled. Set `X-Robots-Tag: noindex, nofollow` server-side.
- **Public report** (`/report/{chat_slug}`): the only indexable surface. Shows redacted aggregates (counts, top phrase categories, daily chart) — never usernames, never message bodies.

This skill applies to public reports only.

## Per-page requirements

Every indexable route needs, set via `@solidjs/meta`:

```tsx
import { Title, Meta, Link } from "@solidjs/meta";

<Title>{t("public-report.meta.title", { chat: chat().title })}</Title>
<Meta name="description" content={t("public-report.meta.description", { chat: chat().title, days: 30 })} />
<Link rel="canonical" href={`https://vixen.example.com/report/${chat().slug}`} />

{/* Open Graph */}
<Meta property="og:type" content="website" />
<Meta property="og:title" content={t("public-report.meta.og-title", { chat: chat().title })} />
<Meta property="og:description" content={t("public-report.meta.og-description")} />
<Meta property="og:image" content={`https://vixen.example.com/report/${chat().slug}/chart.png`} />
<Meta property="og:url" content={`https://vixen.example.com/report/${chat().slug}`} />

{/* Twitter */}
<Meta name="twitter:card" content="summary_large_image" />
<Meta name="twitter:title" content={t("public-report.meta.og-title", { chat: chat().title })} />
<Meta name="twitter:description" content={t("public-report.meta.og-description")} />
<Meta name="twitter:image" content={`https://vixen.example.com/report/${chat().slug}/chart.png`} />
```

## Rules

- **Title**: 50–60 chars, chat name first, brand at the end (`{chat name} — Vixen public report`).
- **Description**: 140–160 chars, factual, one sentence ("Anti-spam moderation summary for {chat} — 30-day rolling stats, no user PII.").
- **Canonical**: always present. Absolute URL, no trailing slash.
- **OG image**: reuse the daily report chart PNG (already 1200×630-ish — adjust if the chart aspect ratio drifts).
- **Every public report must return 200 with the meta in the initial HTML** for crawlers that don't run JS. Means either pre-rendering or serving pre-rendered HTML for bot UAs.
- **No PII in any meta field.** The chat title is OK (it's already public Telegram metadata); chart counts are OK; usernames / message bodies are NOT.

## Structured data (JSON-LD)

```tsx
<script type="application/ld+json">{JSON.stringify({
  "@context": "https://schema.org",
  "@type": "Dataset",
  "name": `Anti-spam stats for ${chat().title}`,
  "description": "Daily aggregated counts of moderation actions, captchas, and verifications.",
  "creator": { "@type": "SoftwareApplication", "name": "Vixen" },
  "license": "https://creativecommons.org/publicdomain/zero/1.0/",
})}</script>
```

Validate with the [Rich Results Test](https://search.google.com/test/rich-results).

## Sitemap + robots

- **`public/robots.txt`**:
  ```
  User-agent: *
  Allow: /
  Allow: /report/
  Disallow: /app/
  Disallow: /api/
  Sitemap: https://vixen.example.com/sitemap.xml
  ```
- **`sitemap.xml`** generated by the server (`GET /sitemap.xml`) lists every public report slug. Cache 24h.
- **`X-Robots-Tag: noindex, nofollow`** header on `/app/*` (authenticated) responses — enforce server-side, don't rely on meta.

## hreflang (RU / EN)

If both locales serve at distinct URLs (`/ru/...` and `/en/...`), every page must list all variants:

```html
<link rel="alternate" hreflang="en" href="https://vixen.example.com/en/report/acme" />
<link rel="alternate" hreflang="ru" href="https://vixen.example.com/ru/report/acme" />
<link rel="alternate" hreflang="x-default" href="https://vixen.example.com/en/report/acme" />
```

If the same URL serves both via `Accept-Language`, skip hreflang and set the canonical to the served URL.

## Core Web Vitals

- **LCP < 2.5s** (chart PNG is the LCP candidate — preload it).
- **INP < 200ms**.
- **CLS < 0.1** (reserve chart space with intrinsic-size).

## Common mistakes

- Same `<title>` on every report (SPA default).
- OG image hardcoded to a generic logo — loses share CTR.
- Canonical pointing to staging URL leaks to prod.
- JSON-LD with broken syntax — use `JSON.stringify`, not template literals.
- Forgetting `noindex` on `/app/*` after a routing refactor — the admin dashboard ends up in Google.
